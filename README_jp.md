# BareFlow  

[English](README.md) | [日本語](README_jp.md)

### Minimal core. Maximum clarity.  
（最小のコアで、最大の明瞭さを。）

BareFlow は、予測可能性・透明性・長期的な保守性を重視するシステムのために設計された、仕様駆動型のフローエンジンです。  
YAML に記述されたフローをそのまま実行し、隠れたロジックや副作用、意図しない挙動を一切持ち込みません。  
コアは徹底的にミニマルでありながら、必要に応じて拡張できる柔軟な構造を備えています。

---

## ✨ BareFlow を選ぶ理由

- **純粋な実行モデル**  
  暗黙的な処理を排した、逐次・決定的なフロー実行。

- **仕様駆動**  
  YAML に書いたとおりに実行される。余計な魔術はない。

- **透明性のある設計**  
  各ステップの実行履歴は StepTrace に明確に記録され、監査やデバッグに強い。

- **ミニマルなコア、拡張可能な外側**  
  Hook / Interceptor / Runtime によって、必要なときだけ機能を追加できる。

- **大規模開発でも破綻しない予測可能性**  
  DSL の複雑さや隠れた状態を排除し、チーム全体で理解しやすい。

---

## 🚀 BareFlow が“何であり、何でないか”

### BareFlow は…
- 純粋な逐次実行エンジン  
- 仕様（Excel/YAML）と実行をつなぐ橋  
- Web/Batch ランタイムの基盤  
- アプリケーションの邪魔をしないフレームワーク

### BareFlow は…ではない
- ワークフローオーケストレーター  
- BPMN エンジン  
- ルールエンジン  
- 自動化ツール  
- 業務ロジックを書く場所  

業務ロジックはあなたのモジュールに閉じ込める。  
BareFlow はただフローを実行するだけ。

## 🧩 コアコンセプト

- **FlowDefinition（YAML）**  
  ロジックを持たない、純粋な実行仕様。

- **ExecutionContext**  
  フロー全体で共有されるデータコンテナ。

- **StepEvaluator**  
  `${key}` を ctx から参照し、最小限の型変換を行う。

- **StepInvoker**  
  モジュールを InputDto → OutputDto で呼び出す。

- **FlowResult**  
  実行結果（status / ctx / stepTrace）を保持。

- **Runtime Layer**  
  Web/Batch との統合、トランザクション、並列化、backoff などを担当。
